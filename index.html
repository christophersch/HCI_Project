    What's the difference between both prototypes?

    In one prototype, you can see your list of tasks (card deck) at all times while performing another task, i.e. when you create a new task, you can still see your existing cards in the background, and removing a task happens in the same view as well.

    In the other prototype, everything takes place on different screens. While creating or removing a task, you don't see the main deck anymore, as these states are new screens that take up the whole view.

    It's a very subtle difference. The experiment mainly tests if users prefer viewing the whole system at once (every action occuring within the same view) while performing different actions, or if this does not really make a difference.

    About Cardinal

    Deadlines are omni-present in our busy day-to-day lives. From the student who rushes an assignment just before midnight, to the employee who needs to finish a project lest they wish to face the wrath of their manager.

    Cardinal is a task-tracking application along the likes of Wunderlist and Trello, but with a simple twist. Instead of letting the user assign only one date to a task, the user inputs two dates: One being the external deadline, for example the 'due by' date on an assignment decided by a professor, the other being an internal deadline, by when the user wishes to get a task done ahead of time. Internal deadlines have been found to help with getting work done (Ariely & Wertenbroch, 2002).



    This research is part of an assignment for the course 'Human Computer Interaction & Affective Computing' at DKE, Maastricht University. Prototypes developed in Proto.io.
